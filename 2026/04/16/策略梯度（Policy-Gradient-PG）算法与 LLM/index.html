策略梯度（Policy Gradient, PG）算法与 LLM

Or1ginnn
2026-04-16
LLM, PG, PPO, 学习笔记

1. 核心思想

直接优化策略

1.1 强化学习基础回顾

  • 智能体（Agent）：与环境交互、学习策略
  • 环境（Environment）：提出状态（State）、奖励（Reward）
  • 策略（Policy）：状态 -> 动作的映射规则
    • 确定性策略：π(a|s) -> 直接输出动作
    • 随机性策略：πθ(a|s) -> 输出动作概率分布（PG 常用） ### 1.2 策略的本质

在强化学习中，策略定义了智能体在状态 s 时选择动作 a 的规则，通常分为： 确定性策略 以及 随机性策略

1.3 参数化意义

通过可学习参数 θ （如神经网络的权重）显示表示策略，使得梯度上升算法可以直接调整参数以优化长期回报。传统方法（如 Q-Learning）间接优化值函数，而策略梯度直接对策略建模，自然支持复杂动作空间（如连续、高维动作）。

1.4 Policy Gradient的核心思想

与其绕道价值函数，不如直接对策略进行参数化，然后通过梯度上升来优化它！

πθ(a|s) 表示一个由参数 θ 定义的策略，它输出在状态 s 下采取动作 a 的概率。目标是：max J(θ) = 𝔼τ ∼ πθ其中 τ 是一条轨迹，R(τ) 是这条轨迹的总奖励。

2. 优势

  1. ==优势一：天然支持连续动作空间==：策略网络可以直接输出连续动作的分布参数（比如高斯分布的均值和方差），从中采样即可得到动作。
  2. ==优势二：可以学习随机策略==：策略直接输出动作的概率分布，天然支持随机性。
  3. ==优势三：更好的收敛性质==：策略参数的小变化只会导致策略分布的小变化，使得优化过程更加平滑稳定。
  4. ==优势四：端到端优化==：直接优化我们真正关心的目标——累积奖励，而不是间接地通过简直函数。

3. Policy Gradient 定理

3.1 推导过程

目标函数：max J(θ) = 𝔼τ ∼ πθ[R(τ)] = ∑τP(τ|θ)R(τ)对 θ 求梯度： ∇θJ(θ) = ∑τ∇θP(τ|θ)R(τ) 这里取巧使用对数导数：∇θP(τ|θ) = P(τ|θ)∇θlog P(τ|θ) 代入得：∇θJ(θ) = ∑τP(τ|θ)∇θlog P(τ|θ)R(τ) = 𝔼τ ∼ πθ[∇θlog P(τ|θ)R(τ)]接下来展开log P(τ|θ)：$$\log P(\tau | \theta) = \log p(s_0) + \sum_{t=0}^{T-1} [\log \pi_\theta (a_t|s_t) + \log p (s_{t + 1} | s_t, a_t)]$$关键观察：p(s0) 和 p(st + 1|st, at) 是环境动态，与 θ 无关！ 因此： $$\nabla_\theta \log P (\tau | \theta) = \sum_{t = 0}^{T - 1} \nabla_\theta \log \pi_\theta(a_t | s_t)$$==最终形式==：Policy Gradinet 定理：$$\nabla_\theta J(\theta) = \mathbb{E}_{\tau \sim \pi_\theta}[\sum_{t=0}^{T-1} \nabla_\theta \log \pi_\theta (a_t|s_t)R(\tau)]$$结果太美了（原文），> 1. 用当前策略采样轨迹 2. 计算每个动作的∇θlog πθ(at|st) 3. 用轨迹奖励加权。

物理意义：$$\nabla_\theta \log \pi_\theta(a_t| s_t) = \frac{\nabla_\theta \pi (a_t|b_t)}{\pi(a_t|s_t)}$$可理解为：==动作概率的相对变化率==，即： - 高概率动作：梯度更新幅度小 - 低概率动作：地图更新幅度大 权重自适应机制：$$\triangle\theta \propto \frac{\nabla_\theta \pi (a_t|b_t)}{\pi(a_t|s_t)} \cdot G_t$$本质是重要性加权更新： - 分母自动调节更新步长 - 避免高概率动作主导优化过程

4. 折扣未来回报Gt

定义：智能体在某个时间步获得的即时奖励，加上未来所有奖励的折扣奖励总和，用于衡量当前状态或动作的长期价值。

4. 1 折扣未来回报数学定义

回报 GT 是从时间步（t）开始的所有的未来奖励的折扣和：$$G_t = R_t + \gamma R_{t+1} + \gamma^2R_{t+2} + \cdots = \sum_{k=0}^{\infty} \gamma^kR_{t+k}$$其中： - Rt: 时间步 t 的即时奖励 - γ：折扣因子，取值范围γ ∈ [0, 1]，用于平衡即时奖励与未来奖励的重要性 ## 4.2 核心思想

  • 即时奖励（Rt）：当前Action带来的直接受益
  • 未来奖励的折扣：未来所有奖励按指数衰减加权求和，体现长期策略的价值。
    • γ 越接近1：模型越重视长期受益（远视）
    • γ 越接近0：模型越关注短期受益（近视）

4.2 为什么需要折扣因子

1. 数学收敛性：无限时间步的奖励求和可能发散，折扣因子确保无穷级数收敛。
2. 不确定性：未来奖励的预测通常不可靠，折扣降低了远期奖励的权重。
3. 实际意义：人类和动物行为也更倾向于即时回报（如“延迟满足”需要更高激励）。

6. 价值函数（V(s)）与贝尔曼方程

6.1 状态价值函数

定义：从状态 s 出发，遵循策略 π 的期望回报。它衡量了处于某个状态的“长期价值”。$$V^\pi(s) = \mathbb{E}[G_t|S_t = s] = \mathbb{E}_\pi[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1} | S_t = s]$$贝尔曼方程的推导： 将 Gt 递归分解为“即时奖励”与“折扣后的未来回报”：Vπ(s) = ∑aπ(a|s)∑s′, rP(s′, r|s, a)[r + γVπ(s′)] ==通俗理解：当前状态的价值 = (即时期望奖励) + (折扣后的下一状态期望价值)==

6.2 分块理解贝尔曼方程

  1. 最外层：按策略选动作 ∑aπ(a|s)
  • π(a|s):在状态 s 下，按策略选动作 a 的概率
  • 求和：把所有可能动作的贡献加起来
  • 意思：对所有可能选的动作，按概率加权平均
  1. 第二层：按环境转移概率看下一步 ∑s′, rP(s′, r|s, a)
  • P(s′, r|s, a):在状态 s 做动作 a ，转移到 s′ 并拿到奖励 r 的概率
  • 求和：把所有可能的下一个状态和奖励加起来
  • 意思：对所有kennel的「下一个状态 + 奖励」，按环境概率加权平均
  1. 最里面的核心：贝尔曼递推关系 [r + γVπ(s′)]
  • r:当前步拿到的即时奖励
  • Vπ(s′):下一个状态 s′ 的价值函数
  • γ:折扣因子，把未来价值打个折
  • 意思：当前的总价值=现在拿的钱 + 未来可能拿的钱的期望

7. LLM中的强化学习

7.1 NLP任务中Agent、Env、State、Action

原文 流程： > - 我们先投喂给模型一个prompt，期望他能产出符合人类喜好哦哦的responsssse > - 在 t 时刻，根据模型上下文，产出一个token，这个token即对应着强化学习中的动作，我们记为Action。因此，不难理解，在NLP语境下，强化学习任务的动作空间就对应着词表 > - 在 t 时刻，模型产出token Action对应着的即时收益为 Rt ，总收益为 Vt （蕴含着即时受益与未来收益两个内容），这个收益即可以理解为“对人类喜好的衡量”。此刻，模型的状态从 St 变为 St + 1，也就是从“上文”变为“上文+新产出token” > - 在NLP语境下，Agent即为LLM模型本身，环境则对应它产出的预料

7.2 RLHF中四个重要角色

如上图，在RLHF-PPO阶段，一共有四个主要模型，分别是： - Actor Model：演员模型，这就是我们想要训练的目标语言模型 - Critic Model：评论家模型，它的作用是估计总收益 Vt - Reward Model：奖励模型，它的作用是计算即时收益 Rt - Reference Mode：参考模型，它的作用是在RLHF阶段给语言模型加一些约束，防止模型训歪（朝不受控制的方向更新，效果可能越来越差）

其中： - Actor/Critic Model： 在RLHF阶段是需要训练的（图中给这两个模型加了粗边）；而Reward/Reference Model是参数冻结的。 - Critic/Reward/Reference Model共同组成了一个“奖励-loss”计算体系，综合他们的结果计算loss，用于更新Actor和Critic Model

7.2.1 Actor Model

Actor Model 就是要训练的目标语言模型。我们一般用 SFT 阶段产出的 SFT 模型来对它做初始化。 本模型设计的目标：让 Actor Model 能产生符合人类喜好的 Response。所以我们的策略是，先喂给 Actor 一条 prompt，让它生成对应的Response。然后，我们再将“prompt + response”送入我们的“奖励-loss”计算体系中去算得最后的loss，用于更新 Actor。

7.2.2 Reference Model（参考模型）

Reference Model： 以下简称 Ref 模型，一般也用 SFT 阶段得到的 SFT 模型做初始化，在RLHF训练过程中，它的参数是冻结的。 Ref 模型的主要作用是防止 Actor 训歪。 “防止模型训歪”换一个更详细的解释是：我们希望训练出来的Actor模型既能达到符合人类喜好的目的，又尽量让它和SFT模型不要差异太大。简言之，我们希望两个模型的输出分布尽量相似。那什么指标能用来衡量输出分布的相似度呢？我们自然而然想到了KL散度[[[#7.2.2.1 KL（KL Divergence） 散度]]。

如图所示： - 对 Actor Model： 我们喂给他一个 prompt，它正常输出对应的 response，那么response中每一个token肯定有它对应的 log_prob 结果，我们把这样的结果记为 log_probs - 对 Ref：把 Actor 生成的 ==“prompt + response”==喂给他，那么它同样能给出每个 token 的 log_prob 结果，我们记其为 ref_log_probs - 上文的概率 probs 表示概率，这个概率就是 LLM 在最后一层 softmax 输出的概率，在reference并不需要输出 token，而是为个个 token 找相对应的概率即可。

工程上的计算方法： - 模型按 πθ​ 生成回答 y1​, y2​, ..., yT​ - 对每个位置 t，同时算：

- 当前模型的概率：$\log \pi_\theta ​(y_t​∣x,y_{1:t−1}​)$
- 参考模型的概率：$\log \pi_{ref} ​(y_t​∣x,y_{1:t−1​})$
  • 单步 KL 近似：KLt​ ≈ log πθ​(yt) − log πref​(yt​)
  • 整句 KL：$KL_{seq} ​= \frac{1}{T} \sum​_{t=1}^T​ KL_t$

​ #### 7.2.2.1 KL（KL Divergence） 散度

衡量两个分布「像不像、差多大」，也叫做相对熵

给定两个离散概率分布 P(x)(真实分布)、Q(x)(近似、预测分布): $$D_{\text{KL}} (P \parallel Q) = \sum_{x} P(x) \log \frac{P(x)}{Q(x)}$$ 连续分布的形式： $$D_{\text{KL}}(P \parallel Q) = \int P(x) \log \frac{P(x)}{Q(x)} \, dx$$

公式拆解： DKL(P ∥ Q) = 𝔼P[log P(x)] − 𝔼P[log Q(x)] - −𝔼P[log P(x)]: 信息熵（Entropy） - −𝔼P[log Q(x)]: 交叉熵（Cross-Entropy）

所以：👉 KL 散度 = 交叉熵 − 信息熵，这也是深度学习里用交叉熵做损失的原因：真实分布熵固定，最小化交叉熵 = 等价于最小化 KL 散度 = 让模型分布贴近真实分布
